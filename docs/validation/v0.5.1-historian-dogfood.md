## Search Errors Result

Before the polish, queries for `v0.4.5`, `in-flight`, `disconnect`, and `delay` returned no matches. The run's persisted review outcome is `pass`, so it was intentionally excluded from failed/needs-fix evidence even though its findings explicitly call the validation a partial pass and describe unverified reconnect behavior.

v0.5.1 adds `review_limitation`. It exposes explicit partial-pass or unverified findings inside pass reviews while retaining `reviewOutcome: pass`, `terminal: true`, and `retryable: false`. It does not classify the v0.4.6 `weak pass` phrase alone as a failure.

## Search Verification Result

Searches for `npm run check` and `git diff --check` with `status: passed` returned the expected historical v0.4.4-v0.4.6 records, exit codes, timestamps, and bounded stdout/stderr. No command was re-executed by the Historian tool.

Dogfood finding: consumers had to subtract timestamps to obtain duration. v0.5.1 adds `durationMs` while keeping the records explicitly historical.

## Search Changed Files Result

The v0.5.0 validation document returned no match because its implementation did not produce an AI Bridge run. v0.5.1 adds `changeOrigin` to clarify created-after-preflight, modified-pre-existing, pre-existing, and unknown evidence.

## Search Reviews Result

`outcome: pass` returned the v0.4.4-v0.4.6 reviews with iteration `1` and `nextIterationAllowed: false`. `query: weak pass` found the v0.4.6 pass review.

## Read-only Boundary

The Historian tools read only AI Bridge-owned history. Dogfood did not:

- start Claude or call the Claude CLI;
- call `ai_bridge_run_verification`;
- execute Git or shell commands from Historian;
- mutate the workspace or historical run/task/transcript/review/verification files;
- scan repository source, `.env`, private keys, or the user home;
- claim historical records prove current workspace state.

v0.5.1 does not implement Tree-sitter, LSP, embeddings, a vector database, code graph, or UI.

## Known Limitations

- Historian only searches persisted AI Bridge history; direct Codex commits have no run evidence.
- Plugin reinstall does not hot-reload the already-running MCP server in the current thread.
- It does not prove current workspace state.
- It cannot return a patch that was not persisted.
- Transcript and snippet scans remain intentionally bounded.
- Ranking is deterministic keyword/relevance ranking, not semantic search.
- No running-state reconnect, live replay, Tree-sitter, LSP, embeddings, graph, index database, or UI is added.

## Follow-up Decision

Retain direct bounded scanning. Future work should first measure whether dense snapshot snippets or missing direct-Codex history materially impair repeated usage before considering new storage or semantic-search architecture.
