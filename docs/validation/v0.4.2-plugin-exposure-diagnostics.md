## Diagnostic Design

`scripts/smoke_mcp_tools.mjs` launches `mcp/server.mjs` directly with Node and an isolated temporary `AI_BRIDGE_HOME`. It sends JSON-RPC `initialize` followed by `tools/list`, checks every required public AI Bridge tool, bounds captured stderr, enforces a five-second timeout, terminates the child process, and removes the temporary directory.

`scripts/diagnose_plugin_exposure.mjs` checks:

- execution from a repository root;
- package, MCP server, and plugin version alignment;
- `.codex-plugin/plugin.json` existence, JSON validity, name, version, skills configuration, and MCP configuration pointer;
- `.mcp.json` existence, JSON validity, `ai-bridge` server entry, command, and `mcp/server.mjs` argument;
- `skills/ai-bridge/SKILL.md`, `mcp/server.mjs`, and `README.md`;
- the isolated MCP smoke result;
- whitespace, non-ASCII, and Windows-special-character path warnings;
- existence-only hints for known local Codex-related directories and configuration paths.

The diagnostic always reports Codex UI/tool exposure as `unknown`. It does not read broad debug logs or infer undocumented Codex installation rules.

## Interpretation Boundary

A passing `npm run smoke:mcp-tools` proves that the local AI Bridge MCP server can initialize and list its registered tools. A passing `npm run diagnose:plugin` proves that the checked local repository and plugin files are internally consistent.

Neither result proves that a fresh ChatGPT/Codex thread will install, discover, connect to, or expose the plugin. If local diagnostics pass while a fresh thread still lacks `ai_bridge_*` tools, the remaining blocker is outside the AI Bridge MCP server runtime path.

## Known Limitations

- Local diagnostics cannot prove Codex plugin exposure in a new thread.
- The smoke validates protocol responses and static tool registration, not tool invocation behavior.
- The smoke does not call the real Claude Code CLI.
- No real Claude dogfood continuation is part of v0.4.2.
- Existing Codex-related paths are reported only as filesystem observations, not proof of active installation or discovery.
