# AI Bridge v0.5.2 Failure Pattern Memory

## Scope

v0.5.2 adds one local read-only MCP tool that extracts reusable failure and validation-risk patterns from bounded AI Bridge-owned history.

## Baseline

Development started from `e3fe7673db190397ff557a9e2592d1381e39bffd`, package/app v0.5.1, plugin `0.5.1+codex.20260629130000`, and a worktree containing only 15 pre-existing `hatch-pet-runs/silver-armor-girl` files.

## Design

The implementation reuses the v0.5.1 Historian loaders, workspace identity, bounded snippets, redaction, corruption diagnostics, and snapshot classification. It performs deterministic aggregation in memory and creates no index or database.

## Tool Added

`ai_bridge_failure_pattern_summary` accepts a required workspace path, an optional time window, a bounded limit, and include flags for passed limitations, verification failures, review limitations, changed-file risks, and preflight risks.

## Pattern Types

The tool emits:

- `verification_failure`
- `review_limitation`
- `failed_or_cancelled_task`
- `changed_file_risk`
- `preflight_baseline_risk`
- `validation_overclaim_risk`

It also returns `topVerificationRisks`, `topChangedFileRisks`, `preflightReminders`, and `reviewChecklist`.

## Read-only Boundary

The tool reads only persisted AI Bridge records under `AI_BRIDGE_HOME`. It does not start Claude, execute verification, invoke Git or shell commands, create runs/tasks, mutate history, or modify a workspace.

## Redaction Boundary

All evidence snippets pass through the existing secret redaction and 500-character bound. Full transcript, stdout, stderr, and patches are not returned. Diagnostics use record identifiers and basenames rather than absolute internal file paths.

## Historical-only Boundary

The result sets `historical_only: true` and `current_workspace_state_not_checked: true`. Failure patterns are reminders, not proof that the current workspace or a future run has the same problem.

## Test Coverage

Focused tests cover failed and timed-out verification aggregation, review limitations inside pass reviews, terminal task statuses, changed-file aggregation, pre-existing baseline noise, invalidated baselines, validation overclaims, include flags, time windows, stable severity ordering, limit clamping, redaction, corruption tolerance, and before/after history and workspace hashes.

## Known Limitations

- The tool does not use semantic search, embeddings, Tree-sitter, LSP, a vector database, SQLite, or a code graph.
- Rules can only summarize evidence that earlier AI Bridge runs persisted.
- It does not inspect or validate current workspace state.
- It does not replace current verification, Run Explorer inspection, or Codex review.
- An already-connected MCP client may require a fresh thread or reconnection after plugin reinstall.

## Follow-up Decision

Retain direct bounded scanning until repeated real usage shows that rule coverage or scan cost requires a narrower follow-up. Do not add a persistent index or semantic architecture without measured need.
