# AI Bridge v0.4.1 Run Explorer Validation

## Scope

This validation covers persisted run discovery, inspection, transcript tailing, baseline-aware diff reporting, historical verification summaries, and redacted JSON/Markdown export. It uses isolated temporary git repositories and fixture state. It does not call the real Claude API.

Baseline before implementation: `5f6332033beb5bff2c7e11e2e9c99b423da959ad`.
Validation-polish starting baseline: `eb392964d2454718c60be799cb97f18534a018ce`.

## Expected Guarantees

- Healthy runs remain discoverable when another run record is corrupt.
- Runs rank by workflow status priority and then newest `updatedAt` within one status.
- Run summaries expose workspace identity, iteration limits, task/session identity, last task status, rank reason, and workspace match.
- Explorer reads isolate corrupt task, transcript, and verification records as diagnostics.
- Transcript tailing uses a stable cursor and does not require a taskId.
- Diff inspection does not write run, task, or workspace state.
- Patch and verification output are opt-in, bounded, and secret-redacted.
- Sensitive filenames are reported through reasoned `sensitivePathWarnings`; `sensitivePaths` remains available for compatibility.
- Historical verification is reported as `passed`, `failed`, `not_run`, or `partial` without executing commands.
- Exports default to AI Bridge storage, omit raw stream-json and patch content by default, and reject overwrite.
- The six Explorer tools do not start Claude or create a new run.

## Automated Coverage

`tests/run-explorer.test.mjs` covers state-priority ranking, product-ready summaries, corrupt run and last-task isolation, workspace filtering, corrupt task/transcript/verification tolerance, terminal final-log repair, cursor tailing, read-only diff behavior, reasoned sensitive path warnings, bounded patch redaction, verification duration/status/output handling, JSON and Markdown reports, default export location, path containment, and overwrite refusal.

Most Run Explorer focused tests use synthetic persisted run fixtures to reduce CI load and isolate read-path behavior. A dedicated smoke test validates Explorer compatibility with a real `preflight()`-created run shape. These tests do not prove real Claude API behavior or full iteration execution.
