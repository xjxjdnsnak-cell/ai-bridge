# AI Bridge v0.5.0 Historian + Workspace Memory Lite

## Scope

v0.5.0 adds local read-only search over persisted AI Bridge workflow history and a compact recent-workspace memory summary. The v0.4.6 in-flight disconnect weak pass is the prerequisite recovery validation.

## Inspiration Boundary

The feature borrows the local-memory principle of avoiding repeated reconstruction of workflow history. It does not implement Tree-sitter, LSP, embeddings, a vector database, source-code semantics, or a code graph.

## Design

Historian directly scans bounded AI Bridge-owned records under `AI_BRIDGE_HOME`. Shared loaders cap file and transcript reads, isolate corrupt data as diagnostics, redact snippets, apply stable relevance ordering, and issue query-bound opaque cursors. v0.5.0 does not create or require a persistent index.

The history path never invokes Claude, verification commands, Git, Run Explorer's live diff collector, or workspace identity routines that shell out to Git.

## Historian Behavior

Run search supports workspace, status, time, text, terminal-state, transcript-event, limit, and cursor filters. Error search covers failed, timed-out, and cancelled tasks; failed verification; needs-fix reviews; transcript errors; and corrupt-record diagnostics. Verification, changed-file, and review searches use only persisted records.

Changed-file search omits patches by default. When requested, it returns only an already persisted patch, bounded to 20 KiB and redacted. If no persisted patch exists, it returns `patch_unavailable` rather than executing `git diff`.

Read-only queries against the local AI Bridge history returned:

- recent runs `run-20260628101649-yhswwz`, `run-20260628054358-6dlpyl`, and `run-20260628052600-7ikdkf`, all `passed` with completed latest tasks;
- cancelled and failed task evidence from older runs through `search_errors`;
- three passed verification records for the v0.4.6 run;
- the v0.4.4, v0.4.5, and v0.4.6 validation document paths through changed-file search;
- three pass reviews with `nextIterationAllowed: false`.

## Workspace Memory Lite Behavior

Workspace Memory Lite summarizes recent runs, changed files, verification commands, failures, and reviews. It derives workspace identity from the normalized input path and reads a repository fingerprint only from stored run metadata. Suggested context explicitly treats the result as historical evidence.

The local example returned workspace key `b23651b0c0f5ba878b8b00a0e075e4321c4ade38a52c11e69f1fd38f8f051975`, stored repository fingerprint `e977cc9477a56b6105e0b21f67b590a1c7bbee827b217f7a5651f1ad3df8ed25`, and three bounded entries in each requested recent-history category.

## Read-only Boundary

Historian and Workspace Memory Lite:

- do not start Claude or call the Claude CLI;
- do not execute verification, Git, or shell commands;
- do not create runs or tasks;
- do not write indexes or rewrite historical records;
- do not read repository source or recursively scan the user home;
- do not mutate the workspace.

Read-only tests hash AI Bridge history and workspace fixtures before and after all search calls and compare file lists and content hashes.

## Redaction Boundary

All snippets are redacted and limited to 500 characters. Transcript scanning is capped at 2,000 events or 1 MiB. Redaction covers `sk-` values, provider API keys, tokens, passwords, secrets, private keys, bearer credentials, and authorization values. Sensitive filenames such as `.env` and `id_rsa` produce warnings without reading their content.

## Corruption Tolerance

Malformed run, task, transcript, verification, review, and snapshot records are isolated as diagnostics. Healthy records remain searchable. Missing optional verification, review, transcript, snapshot, or patch data does not crash the request.

## Test Coverage

Focused tests cover:

## Known Limitations

- v0.5.0 does not address running-state reconnect.
- v0.5.0 does not implement live replay.
- Historian cannot return a patch that was not persisted previously.
- Historical evidence is not proof of current workspace state.
- v0.5.0 does not implement Tree-sitter, LSP, embeddings, a code graph, or UI.
- Direct scanning is intentionally bounded and may report truncation diagnostics for large history files.

## Follow-up Decision

Retain direct bounded scanning for v0.5.0. Consider a deletable, rebuildable index only after measured history volume shows that direct scans are insufficient.
